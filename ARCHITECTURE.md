## 一、核心技术选型（已确定）

### 1.1 开发语言
| 选项 | 选择 | 理由 |
|------|------|------|
| Python | ✅ | AI 生态最成熟，LangChain/CrewAI/AutoGen 全是 Python |
| Go | ❌ | AI 领域生态弱，开发周期长 |
| TypeScript | ❌ | 适合 Web，不适合底层 Agent |

### 1.2 架构模式
| 选项 | 选择 | 理由 |
|------|------|------|
| 单一 Agent | ❌ | 能力有限，复杂任务处理差 |
| **Supervisor + Workers** | ✅ | 成本低 80%，速度快 3 倍 |

### 1.3 推理框架
| 选项 | 选择 | 理由 |
|------|------|------|
| 纯 ReAct | ❌ | 容易失控，AI 乱操作 |
| **ReAct + Plan-Act 双阶段** | ✅ | Supervisor 审批后再执行，稳定 10 倍 |
| ToT（树状思维） | ○ | 仅复杂任务启用，简单任务跳过 |

### 1.4 GUI 操控
| 选项 | 选择 | 理由 |
|------|------|------|
| pyautogui（坐标） | ❌ | 窗口移动就废 |
| **pywinauto（UIA）** | ✅ | 读窗口树，精准，无白名单 |
| Windows UIA 直接调用 | ✅ | 底层 API，最稳定 |

### 3.2 Plan-Act 双阶段详解

```python
# Worker 的工作方式（伪代码）
class Worker:
    async def plan(task) -> ExecutionPlan:
        """
        阶段 1：规划（Plan）
        - 分析任务
        - 生成执行步骤
        - 预估风险
        - 返回计划给 Supervisor
        """
        return ExecutionPlan(
            steps=[...],
            risk_level="medium",  # low / medium / high
            estimated_time=60
        )
    
    async def execute(approved_plan) -> Result:
        """
        阶段 2：执行（Act）
        - 只有收到 Supervisor 批准后才执行
        - 按计划步骤执行
        - 返回结果
        """
        ...
```

### 3.3 风险分级

| 等级 | 操作示例 | 处理方式 |
|------|----------|----------|
| 🟢 低风险 | 读文件、查天气、搜索、聊天 | 直接执行 |
| 🟡 中风险 | 写文件、执行命令、打开应用 | Plan 后确认 |
| 🔴 高风险 | 删文件、系统设置、联网操作 | 拒绝或需密码 |

```markdown
# Woclaw 学习经验库

## 用户偏好
- 喜欢简洁的回答，不要太长
- 文件路径用中文描述

## 常用工作流
- 整理文件：先扫描 → 分类 → 移动
- 写代码：先确认需求 → 写框架 → 补充细节

## 成功案例
### 2026-03-29: 批量重命名文件
- 任务：把下载文件夹的图片按日期重命名
- 方法：先用 dir 命令获取文件列表 → Python 脚本批量处理
- 教训：不要用 PowerShell 的 Rename-Item，容易编码问题

## 最佳实践
- Windows 文件操作优先用 pathlib
- GUI 操控先截图确认界面状态
- 调用外部命令加 timeout，防止卡死
```

### 4.3 ERRORS.md 示例

```markdown
# Woclaw 错误记录

## 已解决
| 时间 | 错误 | 原因 | 解决方案 |
|------|------|------|----------|
| 2026-03-29 | pyautogui 点击失败 | 窗口焦点丢失 | 先激活窗口再用 |
| 2026-03-29 | 文件名含中文移动失败 | 编码问题 | 用 shutil 而非 os.rename |

## 避免方法
1. Windows 文件操作永远用 `pathlib.Path` 或 `shutil`
2. GUI 操作前先调用 `window.activate()`
3. 网络请求加 timeout 和重试
```
